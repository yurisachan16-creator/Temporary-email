# TempMail+ — 临时邮箱浏览器插件

> 版本 v1.0.0 · 支持 Chrome / Firefox / Edge

一键生成临时邮箱，在浮动面板内直接收信、读信、自动填入，无需跳转外部网站。

---

## 两种安装方式对比

| | 篡改猴脚本（推荐） | 浏览器扩展 |
|-|------------------|-----------|
| **安装难度** | ⭐ 超简单：粘贴一个文件即可 | 需要开启开发者模式 |
| **更新方式** | 替换脚本文件 | 重新加载扩展目录 |
| **UI 形态** | 所有网页右下角的浮动面板 | 点击工具栏图标弹窗 |
| **权限** | 无需特殊浏览器权限 | 需要 tabs、storage 等权限 |
| **Firefox 稳定性** | 重启不失效 | 临时加载，重启失效 |

## 1. 功能概览

| 功能 | 说明 |
|------|------|
| **生成临时邮箱** | 一键从 1secmail 获取随机邮箱地址，无需注册 |
| **复制地址** | 点击「复制」将邮箱地址写入剪贴板 |
| **自动填入** | 点击「自动填入」，插件自动识别当前页面的邮箱输入框并填入 |
| **收件箱** | 直接在弹窗内查看收到的邮件列表 |
| **自动轮询** | 每 10 秒自动刷新一次收件箱，收到新邮件有提示 |
| **阅读邮件** | 点击邮件条目，在弹窗内完整阅读邮件正文 |
| **丢弃邮箱** | 用完后一键丢弃，彻底与该地址断开关联 |
| **多语言** | 自动跟随浏览器语言，支持中文和英文 |
| **持久化** | 关闭再打开浏览器后，上次的邮箱地址自动恢复 |

## 3. 安装步骤

---

### 3.1 方式一：篡改猴脚本（推荐）

这是最简单的安装方式，**不需要开发者模式，不需要 Node.js**，5 分钟完成。

#### 第一步：安装篡改猴（Tampermonkey）

在你使用的浏览器中，从官方商店安装 Tampermonkey 扩展：

| 浏览器 | 安装地址 |
|--------|---------|
| Chrome | Chrome 应用商店搜索「Tampermonkey」|
| Edge   | Edge 加载项商店搜索「Tampermonkey」|
| Firefox | Firefox 扩展商店搜索「Tampermonkey」|

安装完成后，浏览器工具栏会出现 Tampermonkey 的图标（黑色圆形）。

#### 第二步：安装 TempMail+ 用户脚本

**方法 A：直接粘贴脚本内容**

1. 打开项目中的 `tampermonkey/TempMail+.user.js` 文件，复制全部内容。
2. 点击浏览器工具栏的 Tampermonkey 图标 → 「**管理面板**」。
3. 点击右上角的「**+**」新建脚本。
4. 将编辑器中的默认内容**全部删除**，粘贴刚才复制的代码。
5. 按 `Ctrl+S` 保存（Mac：`Cmd+S`）。

**方法 B：从文件导入**

1. 点击 Tampermonkey 图标 → 「管理面板」。
2. 点击右上角「**实用工具**」选项卡。
3. 在「从文件导入」区域，点击「**选择文件**」，选择 `tampermonkey/TempMail+.user.js`。
4. 点击「导入」确认。

#### 第三步：验证安装

打开任意网页（例如百度），页面**右下角**会出现一个紫色的 ✉ 悬浮按钮，点击即可展开面板。

---

### 3.2 方式二：Chrome / Edge 扩展

> 需要开启开发者模式，适合开发者使用。

#### 准备项目文件

```bash
# 克隆仓库
git clone git@github.com:yurisachan16-creator/Temporary-email.git
cd Temporary-email

# 安装依赖（同时将 polyfill 和 DOMPurify 复制到 src/lib/）
npm run setup
```

如果 `npm run setup` 报错，可分步执行：

```bash
npm install
node scripts/copy-libs.js
```

#### 加载扩展

1. 在地址栏输入并打开：
   - Chrome：`chrome://extensions`
   - Edge：`edge://extensions`

2. 打开右上角的「**开发者模式**」开关（Chrome 在右上角；Edge 在左侧栏底部）。

3. 点击「**加载已解压的扩展程序**」（Chrome）或「**加载解压缩的扩展**」（Edge）。

4. 在弹出的文件选择框中，找到并选择项目的 **`src/`** 文件夹，点击「选择文件夹」。

5. 扩展图标（✉）将出现在浏览器工具栏。如果看不到，点击工具栏右侧的拼图图标（🧩），找到 TempMail+ 后点击「固定」。

---

### 3.3 方式三：Firefox 扩展

1. 在地址栏输入并打开：`about:debugging#/runtime/this-firefox`

2. 点击「**临时载入附加组件**」。

3. 在弹出的文件选择框中，进入项目的 `src/manifest/` 文件夹，选择 **`manifest.firefox.json`** 文件（注意：Firefox 需选择具体的 JSON 文件，不是文件夹）。

4. 扩展图标（✉）将出现在工具栏。

> **注意**：Firefox 临时加载的扩展在浏览器重启后会失效，需重新加载。推荐使用篡改猴方式，重启不失效。

## 4. 使用说明

> **两种安装方式的操作完全相同**，唯一区别是打开面板的方式：
> - **篡改猴版**：点击页面右下角的紫色 ✉ 悬浮按钮
> - **扩展版**：点击浏览器工具栏的 ✉ 图标

### 第一步：生成临时邮箱

打开面板后，首次使用时会看到「空状态」页面：

```
┌────────────────────┐
│ ✉ TempMail+        │
├────────────────────┤
│         📬         │
│                    │
│  使用临时邮箱保护您  │
│  的真实邮件地址...   │
│                    │
│  [ 生成临时邮箱 ]   │
└────────────────────┘
```

点击「**生成临时邮箱**」按钮，等待约 1 秒，即可看到生成的邮箱地址，例如：

```
abc123@1secmail.com
```

该地址已自动保存，下次打开插件无需重新生成。

---

### 第二步：去目标网站注册

切换到你要注册的网站，找到邮箱输入框，准备填入地址。

---

### 第三步：自动填入邮箱

**方法 A（推荐）— 自动填入**

1. 确保目标网站的注册页面是当前激活的标签页。
2. 点击插件工具栏图标打开弹窗。
3. 点击「**自动填入**」按钮。
4. 插件会自动识别页面上的邮箱输入框，将地址填入。
5. 弹窗底部会短暂出现「邮箱已成功填入 ✓」提示。

**方法 B — 手动复制粘贴**

1. 在弹窗中点击「**复制**」按钮，按钮短暂变为「已复制 ✓」。
2. 到目标网站的邮箱输入框，按 `Ctrl+V`（Mac：`Cmd+V`）粘贴。

> **提示**：如果「自动填入」提示「页面上未找到邮箱输入框」，说明该网站的输入框结构比较特殊，请改用手动复制粘贴。

---

### 第四步：等待并查看邮件

完成网站注册操作后，回到插件弹窗。收件箱会每 **10 秒**自动刷新一次。

收件箱状态指示：

- **绿色闪烁圆点 ●**（标题栏右侧）：自动轮询正在运行
- **「暂无邮件，等待收件中...」**：还没有新邮件，稍等片刻
- 邮件条目显示：发件人 / 主题 / 时间

你也可以随时点击「**刷新**」按钮手动更新收件箱。

收到新邮件时，弹窗底部会出现「**收到新邮件**」提示。

---

### 第五步：阅读邮件详情

在收件箱列表中点击任意一封邮件，进入详情视图：

```
┌────────────────────────┐
│ [← 返回]  邮件主题      │
├────────────────────────┤
│ 发件人: noreply@xx.com  │
│ 时间: 2026-03-26 12:00 │
├────────────────────────┤
│                        │
│  （邮件正文内容）        │
│  包含图片、链接等        │
│                        │
└────────────────────────┘
```

- 点击「**← 返回**」回到收件箱列表。
- 邮件正文支持 HTML 富文本显示（已经过安全处理，危险脚本已被移除）。

---

### 第六步：用完后丢弃邮箱

注册完成、确认不再需要该邮箱后：

1. 点击「**🗑 丢弃**」按钮。
2. 弹出确认对话框：「确认丢弃此邮箱？丢弃后将无法恢复。」
3. 点击「**确定**」，邮箱地址被清除，插件回到空状态，可重新生成。

> **注意**：丢弃仅清除插件本地记录。1secmail 服务器上的邮件会在其自然过期后删除（通常 1 小时至数天不等），无法提前手动删除。

## 6. 常见问题

**Q：生成邮箱时提示「网络异常，请检查连接」？**

A：插件通过公开 API（`www.1secmail.com`）生成邮箱。请检查：
- 网络是否正常（可在浏览器访问 `https://www.1secmail.com`）
- 是否开启了代理或防火墙拦截了该域名

---

**Q：自动填入没有效果，提示「页面上未找到邮箱输入框」？**

A：插件通过以下方式识别邮箱输入框：
- `<input type="email">`
- `<input name="email">` 或 name/id 包含 "email"
- `<input placeholder>` 包含 "email" 或 "邮箱"

部分网站使用自定义组件（如富文本框、Shadow DOM），无法被自动识别。此时请使用「复制」后手动粘贴。

---

**Q：点击「自动填入」后，网站的前端框架（React/Vue）没有响应输入？**

A：插件在填入时会同时触发 `input` 和 `change` 两个原生事件，大多数框架可正常响应。如果仍无效（例如某些特殊框架），请手动点击输入框后按 `Ctrl+V` 粘贴。

---

**Q：关闭浏览器后重新打开，邮箱地址还在吗？**

A：在。插件使用 `browser.storage.local` 持久化存储邮箱地址，关闭再打开浏览器后会自动恢复上次的邮箱，并开始轮询。

---

**Q：重新安装或更新插件后，之前的邮箱还能找回吗？**

A：
- **更新插件**（同一 Extension ID）：本地存储保留，邮箱地址还在。
- **卸载后重装**：本地存储被清除，无法恢复。邮件本身仍在 1secmail 服务器上，但没有地址记录就无法访问。

---

**Q：临时邮箱的邮件能保存多久？**

A：取决于 1secmail 服务。通常邮件在服务器上保留 **1 小时至数天**，之后自动删除。不要用临时邮箱接收重要信息。

---

**Q：Firefox 重启后插件消失了？**

A：这是 Firefox 临时加载的正常现象。推荐改用篡改猴方式安装（见 [3.1](#31-方式一篡改猴脚本推荐)），重启不失效。

---

**Q：篡改猴版的悬浮按钮挡住了页面内容怎么办？**

A：拖动面板标题栏可以将整个面板移到任意位置。关闭面板后悬浮按钮固定在右下角，不影响大多数页面操作。如果特定网站上按钮干扰使用，可以在 Tampermonkey 管理面板中，为该网站单独禁用脚本（右键图标 → 「排除」当前页面）。

---

**Q：篡改猴版提示「DOMPurify 未加载」，邮件正文显示为纯文本？**

A：脚本通过 `@require` 从 jsdelivr CDN 加载 DOMPurify。如果网络无法访问该 CDN，HTML 邮件会降级为纯文本显示（功能正常，样式缺失）。可在脚本编辑器中将 `@require` 的 URL 改为可访问的镜像地址。

---

**Q：插件的邮件语言显示为英文，能改成中文吗？**

A：插件自动读取浏览器界面语言。如果你的 Chrome/Firefox 系统语言设置为英文，插件也会显示英文。修改方法：
- Chrome：设置 → 语言 → 将「中文（简体）」移到第一位
- Firefox：设置 → 语言 → 选择「中文（简体）」

### 关键架构约定

1. **API 调用**：所有 1secmail API 请求必须通过 `src/api/mailService.js`，其他文件不直接 `fetch`。

2. **存储操作**：统一使用 `src/utils/storage.js`，存储键为 `currentEmail`（当前邮箱）和 `emailList`（历史列表，v2.0 使用）。

3. **HTML 消毒**：所有写入 DOM 的邮件 HTML 内容必须先经过 `src/utils/sanitize.js` 的 `sanitizeHTML()` 处理。

4. **跨浏览器 API**：统一使用 `browser.*`（Promise 风格），由 `webextension-polyfill` 在 Chrome/Edge 上抹平差异。

5. **多语言**：新增文案必须同时更新 `zh.json`、`en.json` 和 `translations.js` 三个文件，不在 JS/HTML 中硬编码字符串。

6. **测试约定**：测试用例不可修改，实现代码必须满足现有测试。

### Git 分支策略

```
main          ← 仅接受来自 develop 的合并，打版本 tag
  └── develop ← 集成分支
        ├── feat/xxx  ← 新功能
        └── fix/xxx   ← Bug 修复
```

提交格式遵循 Conventional Commits，示例：
```
feat(popup): 添加邮件详情视图
fix(content): 修复 Shadow DOM 内输入框无法识别的问题
chore(deps): 升级 dompurify 至 3.x
```

## 8. 已知限制

| 限制 | 说明 |
|------|------|
| 无法发送邮件 | 1secmail 仅支持接收，临时邮箱不能发件 |
| 邮件自动过期 | 服务器端邮件会在一定时间后自动删除，无法保存 |
| 不支持附件 | 邮件附件不可下载（API 限制） |
| Firefox 临时加载（扩展版） | 未正式发布，重启浏览器后需重新加载；篡改猴版不受此限制 |
| 浏览器内置页面无法自动填入 | `chrome://`、`about:` 等页面不允许脚本注入 |
| 轮询依赖面板/弹窗开启 | 关闭面板后停止轮询，后台不会继续刷新（v2.0 计划改进） |
| 不支持 HTTPS 拦截的企业网络 | 部分企业防火墙会拦截对 1secmail.com 的请求 |
| DOMPurify 依赖 CDN（篡改猴版） | 首次加载需访问 jsdelivr，离线或受限网络下 HTML 邮件降级为纯文本 |
