## 1. 产品概述

### 1.1 产品名称
**TempMail+**（临时邮箱快速生成插件）

### 1.2 产品定位
一款运行于 Chrome / Firefox / Edge 浏览器的扩展插件，帮助用户在需要填写邮箱地址时，快速生成一个一次性临时邮箱，接收验证码或注册邮件，保护个人真实邮箱不被暴露或滥用。

### 1.3 核心价值
| 痛点 | 解决方案 |
|------|----------|
| 注册网站需要邮箱，不想暴露真实邮件 | 一键生成临时邮箱，隔离个人隐私 |
| 切换网站查收验证码，操作繁琐 | 在插件弹窗内直接查看收件，无需跳转 |
| 邮箱填写麻烦，需手动复制粘贴 | 自动识别并填入页面邮箱输入框 |

---

## 2. 目标用户

| 用户类型 | 使用场景 |
|----------|----------|
| 普通网民 | 注册不重要的网站账号，避免垃圾邮件 |
| 开发者 / 测试人员 | 批量测试注册流程，无需准备多个真实邮箱 |
| 隐私重视者 | 不希望真实邮箱被第三方收集或出售 |

## 4. 功能需求

---

### v0.1 原型版

> 目标：最小可用产品，跑通"生成邮箱 + 展示收件"主链路

#### F-01 生成临时邮箱
- **描述**：用户点击「生成」按钮，从 1secmail API 获取一个随机邮箱地址
- **验收标准**：
  - [ ] 成功调用 API 并展示邮箱地址（格式：`xxx@1secmail.com`）
  - [ ] 邮箱地址可一键复制到剪贴板
  - [ ] 生成失败时展示错误提示

#### F-02 展示收件列表
- **描述**：在弹窗内展示该邮箱收到的邮件列表（发件人、主题、时间）
- **验收标准**：
  - [ ] 列表按时间倒序排列
  - [ ] 无邮件时展示「暂无邮件」空状态
  - [ ] 手动点击「刷新」可重新获取列表

#### F-03 基础弹窗 UI
- **描述**：点击浏览器工具栏图标弹出操作面板
- **验收标准**：
  - [ ] 弹窗宽度 380px，展示邮箱地址和收件列表
  - [ ] 支持中文显示

---

### v1.0 闭环版

> 目标：从生成到读信，用户全流程无需离开插件

#### F-04 邮件详情阅读
- **描述**：点击列表中的邮件，展示邮件正文内容
- **验收标准**：
  - [ ] 支持 HTML 格式邮件渲染（需 sanitize，防 XSS）
  - [ ] 支持纯文本邮件展示
  - [ ] 提供「返回列表」按钮

#### F-05 自动填入页面输入框
- **描述**：检测当前页面中的邮箱输入框，自动或一键填入临时邮箱地址
- **验收标准**：
  - [ ] 能识别 `input[type="email"]` 及 `input[name*="email"]` 等常见选择器
  - [ ] 填入前弹出确认提示（可在设置中关闭）
  - [ ] 填入失败时提示用户手动复制

#### F-06 自动轮询收件
- **描述**：生成邮箱后，插件后台每 10 秒自动检查一次新邮件
- **验收标准**：
  - [ ] 轮询期间展示「等待收件...」状态
  - [ ] 收到新邮件时列表自动更新
  - [ ] 关闭弹窗后停止轮询（节省资源）

#### F-07 邮箱地址持久化
- **描述**：使用 `browser.storage.local` 保存当前邮箱，关闭弹窗后重新打开仍可使用
- **验收标准**：
  - [ ] 重新打开弹窗后自动恢复上次的邮箱地址
  - [ ] 提供「丢弃此邮箱」按钮，手动清除

#### F-08 多语言支持（中/英）
- **描述**：插件 UI 支持中文与英文，跟随浏览器语言自动切换，也可手动切换
- **验收标准**：
  - [ ] 默认语言：中文
  - [ ] 浏览器语言为英文时自动使用英文
  - [ ] 弹窗内提供语言切换入口
  - [ ] 所有 UI 文本均通过 i18n 文件管理，不硬编码

#### F-09 多浏览器兼容
- **描述**：插件在 Chrome、Firefox、Edge 三个浏览器上均可正常安装和使用
- **验收标准**：
  - [ ] 使用 `webextension-polyfill` 统一 API 调用
  - [ ] Chrome / Edge 使用 Manifest V3
  - [ ] Firefox 使用 Manifest V2（兼容层）或 MV3（≥109）
  - [ ] 三端功能一致，UI 无明显差异

### v2.0 优化版

> 目标：功能完整、体验精良，可公开发布到插件商店

#### F-10 多邮箱管理
- **描述**：允许用户同时保存多个临时邮箱地址，切换查看各自的收件
- **验收标准**：
  - [ ] 最多同时保存 5 个邮箱
  - [ ] 支持为每个邮箱备注命名
  - [ ] 支持单独删除某个邮箱

#### F-11 新邮件桌面通知
- **描述**：收到新邮件时，通过浏览器通知 API 弹出桌面提醒
- **验收标准**：
  - [ ] 通知内容包含发件人和主题
  - [ ] 点击通知可打开插件弹窗并定位到该邮件
  - [ ] 可在设置中开关此功能

#### F-12 邮箱有效期倒计时
- **描述**：展示临时邮箱的预估有效时间，到期前提醒用户
- **验收标准**：
  - [ ] 弹窗内显示倒计时（基于 1secmail 约 1 小时有效期）
  - [ ] 到期前 10 分钟展示黄色警告
  - [ ] 到期后提示「邮箱已失效，请重新生成」

#### F-13 深色模式
- **描述**：跟随系统或手动切换深色 / 浅色主题
- **验收标准**：
  - [ ] 支持 `prefers-color-scheme` 自动切换
  - [ ] 可在设置中手动覆盖

#### F-14 邮件内容搜索
- **描述**：在收件列表中按关键词搜索邮件
- **验收标准**：
  - [ ] 支持按发件人、主题搜索
  - [ ] 实时过滤，无需提交

#### F-15 上架插件商店
- **描述**：打包并提交到 Chrome Web Store、Firefox Add-ons、Edge Add-ons
- **验收标准**：
  - [ ] 符合各商店的隐私政策要求
  - [ ] 提供完整的商店截图和描述文案

## 5. 技术架构

### 5.1 目录结构（规划）

```
Temporary-email/
├── src/
│   ├── manifest/
│   │   ├── manifest.chrome.json    # Chrome / Edge MV3 配置
│   │   └── manifest.firefox.json  # Firefox MV2/MV3 配置
│   ├── popup/
│   │   ├── popup.html              # 弹窗页面
│   │   ├── popup.js                # 弹窗逻辑
│   │   └── popup.css               # 弹窗样式
│   ├── content/
│   │   └── content.js              # 内容脚本：自动填入邮箱
│   ├── background/
│   │   └── background.js           # 后台脚本：轮询收件
│   ├── api/
│   │   └── mailService.js          # 封装 1secmail API 调用
│   ├── i18n/
│   │   ├── zh.json                 # 中文语言包
│   │   └── en.json                 # 英文语言包
│   └── utils/
│       ├── storage.js              # 封装 browser.storage 操作
│       └── sanitize.js             # HTML 邮件内容消毒（防 XSS）
├── icons/                          # 插件图标（16/48/128px）
├── PRD.md                          # 本文档
├── CLAUDE.md                       # Claude Code 工作指引
├── CHANGELOG.md                    # 版本变更记录
└── README.md                       # 项目介绍
```

### 5.2 技术选型

| 技术点 | 选型 | 原因 |
|--------|------|------|
| 框架 | 原生 JS（无框架） | 插件体积小、无构建依赖、易于调试 |
| 跨浏览器 API | `webextension-polyfill` | 统一 `browser.*` API，减少兼容代码 |
| 邮件 HTML 渲染 | DOMPurify | 成熟的 XSS 防护库 |
| 样式 | 原生 CSS Variables | 支持主题切换，无需预处理器 |
| 打包 | 无打包（直接加载） / 可选 webpack | 原型阶段无需打包 |
| 版本管理 | Git + GitHub | 分支策略见下节 |

### 5.3 Git 分支策略

```
main            ← 稳定发布版本，打 tag（v0.1.0 / v1.0.0 / v2.0.0）
  └── develop   ← 日常开发集成分支
        ├── feat/generate-email     ← 功能分支
        ├── feat/auto-fill
        ├── fix/polling-memory-leak ← 修复分支
        └── chore/i18n-setup        ← 杂项分支
```

**提交信息规范**（Conventional Commits）：

## 6. API 规格

### 6.1 使用服务：1secmail（免费，无需 Key）

**Base URL**：`https://www.1secmail.com/api/v1/`

| 功能 | 方法 | 接口 |
|------|------|------|
| 生成随机邮箱 | GET | `?action=genRandomMailbox&count=1` |
| 获取收件列表 | GET | `?action=getMessages&login={login}&domain={domain}` |
| 读取邮件详情 | GET | `?action=readMessage&login={login}&domain={domain}&id={id}` |
| 删除邮件 | GET | `?action=deleteMessage&login={login}&domain={domain}&id={id}` |

**可用域名列表**（动态获取）：
```
GET ?action=getDomainList
```

### 6.2 错误处理规范

| 场景 | 处理方式 |
|------|----------|
| 网络请求失败 | 展示「网络异常，请检查连接」，提供重试按钮 |
| API 返回空数组 | 展示「暂无邮件」空状态 |
| API 限流（429） | 停止轮询，展示「请求过于频繁，稍后重试」 |
| 邮箱已过期 | 提示「邮箱已失效」，引导重新生成 |

## 8. UI 规格

### 8.1 弹窗尺寸
- 宽度：`380px`（固定）
- 高度：最小 `200px`，最大 `560px`（超出滚动）

### 8.2 主要视图

| 视图 | 触发条件 | 主要元素 |
|------|----------|----------|
| **首页（无邮箱）** | 首次安装或丢弃邮箱后 | 「生成临时邮箱」按钮 |
| **主视图（有邮箱）** | 已有邮箱时 | 邮箱地址 + 复制按钮 + 收件列表 |
| **邮件详情视图** | 点击某封邮件 | 返回按钮 + 邮件正文 |

## 9. 非功能需求

| 指标 | 要求 |
|------|------|
| 插件包体积 | < 500KB（不含 polyfill） |
| 弹窗首次渲染 | < 300ms |
| API 请求超时 | 10 秒后自动失败并提示 |
| 轮询间隔 | 10 秒（避免 API 限流） |
| 内存占用 | 后台脚本空闲时不超过 10MB |
| 安全 | 所有渲染的 HTML 邮件内容必须经过 DOMPurify 处理 |

## 10. 风险与约束

| 风险 | 影响 | 应对方案 |
|------|------|----------|
| 1secmail API 不稳定或关闭 | 核心功能失效 | 预留多 API 切换机制（v2.0 支持备用服务） |
| 浏览器 MV3 迁移政策变化 | Firefox 兼容性问题 | 持续关注 MDN 和各浏览器发布说明 |
| 临时邮箱被目标网站屏蔽 | 用户无法用于某些网站 | 文档说明，不在插件层面绕过 |
| XSS 攻击（恶意邮件内容） | 安全漏洞 | 强制使用 DOMPurify，sandbox iframe 渲染 |
